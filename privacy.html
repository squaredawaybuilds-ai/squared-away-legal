1. Scope

This policy describes how the Squared Away QuickBooks Connector (the "Software") handles data. The Software is a private, internal-use integration operated by Squared Away Builds ("we", "us") to access our own QuickBooks Online company data. It is not offered to third parties.

2. What data we access

When you authorize the Software via Intuit's OAuth 2.0 flow, the Software is granted the com.intuit.quickbooks.accounting scope. Through this scope, it can read and (when explicitly invoked) write the standard accounting entities of the authorized QuickBooks Online company, including:

  • Company profile and preferences
  • Customers and vendors
  • Invoices, estimates, bills, and payments
  • Products, services, and the chart of accounts
  • Tax codes and tracking dimensions (classes, locations, projects)
  • Standard accounting reports (P&L, Balance Sheet, A/R and A/P aging, sales summaries)
  • Employees and time-activity records, where present

The Software does not request the QuickBooks Payments scope and cannot move money, process payments, or initiate transfers.

3. Where data is stored

  • OAuth tokens: Stored locally on your own device at ~/.quickbooks-mcp/tokens.json with restricted file permissions (mode 0600, owner-readable only).
  • Application credentials: Your Intuit Developer Client ID and Client Secret are stored locally in a .env file at the application directory with the same restricted permissions, and are excluded from any version-control repository.
  • QuickBooks data: Not persisted by the Software. Data is fetched from Intuit's API on-demand and held only in memory for the duration of a single request.

4. Who has access

Only the operator (a single individual at Squared Away Builds) has access to the device on which the Software runs and to the locally stored tokens and credentials. No data is shared with any third party other than Intuit, which is the source of the data.

5. Third-party services

The Software communicates exclusively with Intuit's QuickBooks Online API at quickbooks.api.intuit.com and Intuit's OAuth 2.0 endpoints at oauth.platform.intuit.com and appcenter.intuit.com. No analytics, telemetry, advertising, or third-party tracking services are used.

6. Data retention

OAuth refresh tokens remain on disk until you revoke authorization in QuickBooks Online or delete the local token file. The Software auto-rotates refresh tokens as Intuit issues new ones. Beyond that, the Software retains no QuickBooks data.

7. Your rights

Because the Software is operated by you for your own data, you have complete control. You may:

  • Revoke authorization at any time via QuickBooks Online (Settings → Apps → Connected → Disconnect).
  • Delete ~/.quickbooks-mcp/tokens.json to remove stored authorization locally.
  • Remove the Software entirely by deleting its installation directory.

8. Security

Tokens and credentials are stored with restricted (owner-only) file permissions. All communication with Intuit occurs over HTTPS. The Software runs locally and does not expose any network listener except a transient localhost-only port (8765) during the one-time OAuth authorization flow.
