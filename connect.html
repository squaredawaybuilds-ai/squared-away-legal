Connect (or reconnect) the Connector to QuickBooks Online

The Connector is authorized using Intuit's standard OAuth 2.0 flow. This is the same flow that authorizes any QuickBooks Online integration; the only difference is that the Connector is operated locally by Squared Away Builds for its own internal use, so the "user" and the "operator" are the same legal person.

To connect for the first time

  1. Ensure the Connector is installed on the device that will run it.
  2. From a terminal, in the Connector's installation directory, run: 
    uv run quickbooks-mcp-setup
  3. The setup tool opens the operator's default web browser to Intuit's OAuth 2.0 authorization page at appcenter.intuit.com.
  4. The operator signs in to QuickBooks Online (Canada) with the credentials for the company they want to authorize, selects the company, and clicks Connect.
  5. Intuit redirects the browser back to a one-time-use callback URL on the operator's own device (http://localhost:8765/callback). The setup tool captures the authorization code, exchanges it for an access token and refresh token, and stores them locally with owner-only file permissions.
  6. The setup tool prints a confirmation. The Connector is ready to use.

To reconnect

Reconnection follows the same steps. Re-running the setup command will replace any previously stored tokens. Common reasons to reconnect:

  • The refresh token has expired (Intuit's refresh tokens live approximately 100 days; the Connector rotates them automatically each time it makes a call, so this is rare).
  • Authorization was revoked in QuickBooks Online.
  • Switching the authorized company.

Where credentials are stored

  • Intuit Developer Client ID + Client Secret: .env in the Connector's installation directory (owner-only, mode 0600).
  • OAuth refresh token + access token: ~/.quickbooks-mcp/tokens.json on the operator's device (owner-only, mode 0600).
